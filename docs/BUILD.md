# Legends of Aetheria — Build & Run Instructions

## Prerequisites

### 1. Install devkitPro

**Linux / macOS:**
```bash
# Download the devkitPro installer
curl -L https://apt.devkitpro.org/install-devkitpro-pacman | sudo bash

# Install 3DS toolchain + citro2d
sudo dkp-pacman -S 3ds-dev
```

**Windows:**
- Download and run the graphical installer from https://devkitpro.org/wiki/Getting_Started
- Select "3DS Development" during installation.

After installation, ensure these environment variables are set:
```bash
export DEVKITPRO=/opt/devkitpro
export DEVKITARM=$DEVKITPRO/devkitARM
export PATH=$DEVKITARM/bin:$DEVKITPRO/tools/bin:$PATH
```

On Linux/macOS, source the profile once:
```bash
source /etc/profile.d/devkit-env.sh
```

### 2. Install Citra (for emulator testing)

- Download from: https://citra-emu.org/download/
- Or use the Lime3DS fork: https://github.com/Lime3DS/Lime3DS

### 3. Install Python + Pillow (for asset generation only)

```bash
pip install Pillow
```

## Step 1 — Generate Placeholder Assets

The game can run without textures (falls back to colored rectangles), but to
test the full asset pipeline, run:

```bash
cd LegendsOfAetheria
python3 tools/generate_placeholder_assets.py
```

This creates `assets/gfx/grass.png`, `wall.png`, and `player.png`.

### Pack into a .t3x sprite sheet

tex3ds is included in devkitPro (`$DEVKITPRO/tools/bin/tex3ds`).

```bash
tex3ds -f rgba8888 -z auto \
  --atlas assets/gfx/grass.png \
          assets/gfx/wall.png \
          assets/gfx/player.png \
  -o romfs/gfx/tiles.t3x
```

**Important:** The order of images in this command determines their index in
the sprite sheet. The Renderer expects:
- Index 0 → grass.png
- Index 1 → wall.png
- Index 2 → player.png

Do not reorder them without updating `Renderer.cpp`.

## Step 2 — Build

From the project root:

```bash
make
```

On success, two output files are created:
- `LegendsOfAetheria.3dsx` — Homebrew executable (for Citra and 3DS homebrew)
- `LegendsOfAetheria.elf`  — Debug ELF (for GDB)

### Clean build

```bash
make clean && make
```

### Common build errors

| Error | Fix |
|---|---|
| `DEVKITPRO not set` | Run `source /etc/profile.d/devkit-env.sh` |
| `cannot find -lcitro2d` | Run `sudo dkp-pacman -S 3ds-dev` |
| `fatal error: 3ds.h not found` | Confirm `$DEVKITPRO/libctru/include` exists |
| `romfsInit()` crash | Ensure `romfs/` folder exists even if empty |

## Step 3 — Run in Citra

1. Open Citra.
2. Go to **File → Load File**.
3. Select `LegendsOfAetheria.3dsx`.

Controls in Citra:
| In-game | Keyboard default |
|---|---|
| D-Pad | Arrow keys |
| Circle Pad | WASD |
| START | Enter |

The top screen shows:
- Green tiles (grass) and brown tiles (walls), or colored rectangles if no .t3x
- A red player sprite
- FPS counter (top-left)

---

## Step 4 — Run on Physical 3DS Hardware

### Homebrew Launcher method (most common)

1. Ensure your 3DS has Homebrew Launcher installed (see https://3ds.hacks.guide).
2. Create the folder `/3ds/LegendsOfAetheria/` on your SD card.
3. Copy `LegendsOfAetheria.3dsx` to `/3ds/LegendsOfAetheria/LegendsOfAetheria.3dsx`.
4. If you built the .t3x, ensure `romfs/gfx/tiles.t3x` is embedded (the Makefile
   passes `--romfs=$(CURDIR)/romfs` to `3dsxtool` automatically).
5. Launch Homebrew Launcher on the 3DS and select the game.

## Debug Builds

The Makefile compiles in debug mode by default (`-g -Wall`).
`LOG(...)` messages print to stdout, visible in Citra's terminal window.

To build a release (stripped, optimized, no log output):

```bash
make CXXFLAGS="-O2 -DNDEBUG -mword-relocations -ffunction-sections -fdata-sections -march=armv6k -mtune=mpcore -mfloat-abi=hard -mtp=soft -DARM11 -D_3DS"
```
